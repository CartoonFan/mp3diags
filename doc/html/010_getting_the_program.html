Getting MP3 Diags
Currently the program is available in source format only, at ttt0 . It is likely that binaries will be made available for some Linux distributions, but I have no experience building RPMs or other kinds of packages, so I can't make any estimation for when the binaries might be available. It seems that openSUSE Build Service is something that might help, but I haven't looked at it. Well, this is an area where I could use some help, if anybody wants to contribute.
Requirements
The development system is openSUSE 10.3, and I also compiled MP3 Diags on openSUSE 11.1 and it seemed to work fine. I expect most Linux distributions from 2007 or newer to be able to compile and run it. Based on what I know, porting to Windows should be rather trivial as long as MinGW is used, but currently the program should trigger assertion failures (assuming that it compiles at all.) Porting to MSVC is not something that I plan to do. It involves more work than MinGW.
In order to compile the project, these packages should be installed:
  • g++
  • libqt4-devel (Qt 4.3.1; something above should be OK too, but there may be some UI glitches, about which I'd like to know)
  • zlib-devel
  • boost-devel
(Well, this is how the packages are called on openSUSE 10.3; they may have sligtly different names, e.g. g++, libqt4-dev, zlib1g-dev and boost-dev on Ubuntu 9.04)
If you find some other required package, please let me know.
Compiling & installing
After unpacking the surce to some directory, just run Install.sh in that directory. Note that double-clicking the script in some file browser may work or not, depending on the "current directory" being set properly and on whether a terminal window gets opened so you can type the root password to get the executable copied to /usr/local/bin. Other options are to manually run the 3 commands in Install.sh, or to compile the project from KDevelop 3.5, if you have it.
Uninstallation can be done through Uninstall.sh. You probably created some data files, and if you are sure you no longer need them you should remove them manually. Note that the data files come in pairs. Although you only specify an ".ini" file when you start the program for the first time (or when you create a new session ttt0 link), two files get created. The ".ini" only stores settings, while the actual data is stored in a file in the same directory and with the same name but with the extension ".dat".
